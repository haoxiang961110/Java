2.String类

2.1String类概述【理解】

​ String 类代表字符串，Java 程序中的所有字符串文字（例如“abc”）都被实现为此类的实例。也就是说，Java 程序中所有的双引号字符串，都是 String 类的对象。String 类在 java.lang 包下，所以使用的时候不需要导包！

2.2String类的特点【理解】

  • 字符串不可变，它们的值在创建后不能被更改
  • 虽然 String 的值是不可变的，但是它们可以被共享
  • 字符串效果上相当于字符数组( char[] )，但是底层原理是字节数组( byte[] )

2.3String类的构造方法【记忆】

  • 常用的构造方法

    方法名 说明
    public String() 创建一个空白字符串对象，不含有任何内容
    public String(char[] chs) 根据字符数组的内容，来创建字符串对象
    public String(byte[] bys) 根据字节数组的内容，来创建字符串对象
    String s = “abc”; 直接赋值的方式创建字符串对象，内容就是abc

2.4创建字符串对象两种方式的区别【理解】

  • 通过构造方法创建

    ​ 通过 new 创建的字符串对象，每一次 new 都会申请一个内存空间，虽然内容相同，但是地址值不同

  • 直接赋值方式创建

    ​ 以“”方式给出的字符串，只要字符序列相同(顺序和大小写)，无论在程序代码中出现几次，JVM 都只会建立一个 String 对象，并在字符串池中维护

1. 类和对象

1.1 类和对象的理解【理解】

客观存在的事物皆为对象 ，所以我们也常常说万物皆对象。

  • 类
    • 类的理解
      • 类是对现实生活中一类具有共同属性和行为的事物的抽象
      • 类是对象的数据类型，类是具有相同属性和行为的一组对象的集合
      • 简单理解：类就是对现实事物的一种描述
    • 类的组成
      • 属性：指事物的特征，例如：手机事物（品牌，价格，尺寸）
      • 行为：指事物能执行的操作，例如：手机事物（打电话，发短信）
  • 类和对象的关系
    • 类：类是对现实生活中一类具有共同属性和行为的事物的抽象
    • 对象：是能够看得到摸的着的真实存在的实体
    • 简单理解：类是对事物的一种描述，对象则为具体存在的事物

1.2 类的定义【应用】

类的组成是由属性和行为两部分组成

  • 行为：在类中通过成员方法来体现（和前面的方法相比去掉static关键字即可）

3. 成员变量和局部变量

3.1 成员变量和局部变量的区别【理解】

  • 类中位置不同：成员变量（类中方法外）局部变量（方法内部或方法声明上）
  • 内存中位置不同：成员变量（堆内存）局部变量（栈内存）
  • 生命周期不同：成员变量（随着对象的存在而存在，随着对象的消失而消失）局部变量（随着方法的调用而存在，醉着方法的调用完毕而消失）
  • 初始化值不同：成员变量（有默认初始化值）局部变量（没有默认初始化值，必须先定义，赋值才能使用）

4.5 封装思想【理解】

  1. 封装概述
    是面向对象三大特征之一（封装，继承，多态）
    是面向对象编程语言对客观世界的模拟，客观世界里成员变量都是隐藏在对象内部的，外界是无法直接操作的
  2. 封装原则
    将类的某些信息隐藏在类内部，不允许外部程序直接访问，而是通过该类提供的方法来实现对隐藏信息的操作和访问
    成员变量private，提供对应的getXxx()/setXxx()方法
  3. 封装好处
    通过方法来控制成员变量的操作，提高了代码的安全性
    把代码用方法进行封装，提高了代码的复用性

6. 继承

6.1 继承的实现（掌握）

  • 继承的概念

    • 继承是面向对象三大特征之一，可以使得子类具有父类的属性和方法，还可以在子类中重新定义，以及追加属性和方法

  • 实现继承的格式

    • 继承通过extends实现
    • 格式：class 子类 extends 父类 { }
      • 举例：class Dog extends Animal { }

  • 继承带来的好处

    • 继承可以让类与类之间产生关系，子父类关系，产生子父类后，子类则可以使用父类中非私有的成员。

  • 示例代码

6.2 继承的好处和弊端（理解）

  • 继承好处
    • 提高了代码的复用性(多个类相同的成员可以放到同一个类中)
    • 提高了代码的维护性(如果方法的代码需要修改，修改一处即可)
  • 继承弊端
    • 继承让类与类之间产生了关系，类的耦合性增强了，当父类发生变化时子类实现也不得不跟着变化，削弱了子类的独立性
  • 继承的应用场景：
    • 使用继承，需要考虑类与类之间是否存在is..a的关系，不能盲目使用继承
      • is..a的关系：谁是谁的一种，例如：老师和学生是人的一种，那人就是父类，学生和老师就是子类

7.6 方法重写（掌握）

  • 1、方法重写概念
    • 子类出现了和父类中一模一样的方法声明（方法名一样，参数列表也必须一样）

    • 当子类需要父类的功能，而功能主体子类有自己特有内容时，可以重写父类中的方法，这样，即沿袭了父类的功能，又定义了子类特有的内容

1.数组内存图

1.1单个数组的内存图

​ 程序开始运行,main方法首先加载到栈内存,在栈内存开辟空间,然后开始加载main方法里的内容,首先是一个数组对象int[] arr的创建,数组的对象名int[] arr会加载进入main方法所在栈内存的空间,而new的数组对象new int[3]会在堆内存开辟空间,有一个地址,开始进行数据的初始化,这里由于运用静态初始化,且数组长度为3,堆内存里数组对象所创建的空间将会被分为三块,由索引来指向数组中具体的数据值,将来调用数组内具体位置的数据也是通过索引,索引从0开始,由于数组是静态初始化,系统将会默认给每个索引对应的数据赋值,整数数据为0,浮点数数据为0.0,字符数据为空字符,字符串数据为null,布尔类型数据为false,此时初始化完成,堆内存的对象将地址赋给main方法里的数组对象名.

​ 程序继续往下执行,输出对象名,即为堆内对象传过来的地址,输出数组中索引为0的数据值,即通过栈内存中的地址找到堆内存中的对象,根据索引,获取相对应的数据值,输出数组中索引为1,2,的数据值过程同上.

​ 最后main方法执行完成,从栈内存消失(弹栈),而堆内存里的对象不会立即消失,等待Jvm垃圾处理器前来清除.

1.2两个数组的内存图

​ 程序开始运行,main方法首先加载到栈内存中,开辟空间,接着程序往下执行,创建了一个对象,对象名留在栈内存,而new的数组对象在堆内存中开辟空间,初始化数组中的数据,根据长度生成索引,有值根据对应索引赋值,若没有赋值则会给出默认值,最后将栈内存的地址赋给对象名,第二个对象的创建也是一样,但是两个对象在堆内存创建的是两个不同的空间.

​ 程序继续运行,我们开始修改第一个对象对应索引的数据值,通过对象名的地址我们访问堆内存的地址,通过索引修改数据对应元素值,最后我们开始输出第一个数组对象,即为第一个数组对象的地址值,然后输出对应索引的数据值,由于我们修改过,这里就是我们修改过的数据值,继续输出第二个对象的地址值,由于第二个对象我们并没有进行赋值,只是进行初始化,所以最后我们输出的值只是系统默认给出的对应数据类型的默认值.

​ 最后程序执行结束,main方法弹栈,堆内存的对象等待jvm的清理.

1.3两个数组指向相同的内存图

​ 程序开始运行,main方法加载到栈内存,在栈内存开辟空间,程序继续运行,数组对象开始创建,对象名留在栈内存,而对象会在堆内存开辟空间,对数组数据进行初始化,最后将堆内存的地址赋给栈内存的对象名.

​ 程序继续运行,我们开始通过栈内存的地址访问堆内存的地址,找到对象创建的空间,并根据索引对对应的数据值进行修改,这时候输出的对象名即为堆内存地址,数组中索引对应的数据也是被修改后的数据值

​ 这时候我们把第一个对象堆内存的地址给第二个变量,第二个变量要修改对应索引的数据值,这时候也是通过栈内存的地址访问堆内存,访问的是第一个对象创建的空间,修改的自然也是第一个数组中对应索引的数据值,这时候我们输出两个对象的地址,由于第二个对象的地址是第一个对象赋予的,所以一样,而第一个对象在堆内存里的数据值已经被第二个对象修改过,所以后来输出对应索引的数据值,应该都是由第二个对象修改后的结果.

​ 最后程序执行结束,main方法弹栈,堆内存的对象等待jvm的清理.

​ 程序开始执行,main方法进栈并开辟空间,接下来我们new了3个String对象,对象名将会在栈内存出现,而String对象则会在堆内存开辟3个不同的空间,并进行数据初始化,由于字符串常量池的存在,数据将会被加载到字符串常量池中,赋予一个地址,且相同的字符串内容在常量池中只会存在一个,堆里对String初始化存放的并不是字符串数据,而是字符串数据在常量池中的地址,初始化完成,堆内存对象创建的空间地址将会被赋予栈内存的对象名.

​ 此时我们将s1和s2是否相等的结果输出,由于对象名存储的是对象在堆里的地址,所以输出的是false,s1和s3的比较也是如此.

​ 这时我们直接赋值3个String变量,其内容一样,直接指向的是常量池,由于常量池中一个数据只会有一个地址,所以这3个变量的地址值一样,我们将他们进行对比,输出的就应该是true.

​ 最后main方法执行结束,从栈内存消失,堆内存对象等待jvm清理

​ ==Java基本类型的包装类的大部分都实现了常量池技术,即Byte,Short,Integer,Long,Character,Boolean;这5种包装类默认创建了数值[-128,127]的相应类型的缓存数据,但是超出此范围仍然会去创建新的对象.==

​ ==两种浮点数类型的包装类Float,Double并没有实现常量池技术.==

6.super内存图

​ 程序开始运行,Demo.class被加载到方法区,其中的main方法也被加载到方法区,并拥有了一个方法区地址,Fu.class被加载到方法区,其中的成员变量也被加载到方法区,由于类中已经赋值,此时方法区的成员变量age的值便为40,Zi.class被加载到方法区,其中的成员变量和成员方法也被加载到方法区,,由于类中已经赋值,此时方法区的成员变量age的值便为20,成员方法被加载,拥有了一个方法区地址.

​ 程序继续运行,main方法进栈,Zi类对象z被创建,对象在堆内存开辟空间,并初始化成员变量和成员方法,由于Zi继承Fu,会先初始化父类的成员变量age,并赋值为40,之后初始化自己的成员变量age,并赋值为20,初始化自己的成员方法,得到该方法在方法区的地址,初始化完成后,把堆内存的地址赋予栈内存的对象名.

​ 程序继续运行,Zi类对象z调用成员方法show,此时通过栈内存地址访问堆内存,再从堆内存中的地址访问方法区,最终调用方法,show方法加载到栈内存,开辟新空间,首先对局部变量age赋值为30,这时我们输出age就是局部变量的值,之后我们要输出this.age,通过栈内存地址找到堆内存,并找到Zi类中成员变量age的值并输出,输出super.age也是一样,通过栈内存的地址访问堆内存,找到其父类的成员变量age的值并输出.

​ 最后main方法执行结束,从栈内存消失,堆内存对象等待jvm清理

7.static内存图

​ ==类在方法区加载完成后也会有一个在方法区的地址,并赋予给类名.==

​ 程序开始运行,Demo05.class被加载到方法区,其中的静态方法main被加载到静态区,完成初始化,并有了一个方法区的地址,StaticaStudent.class被加载到方法区,其中的静态成员变量university和静态方法show02()被加载到静态区完成初始化,静态成员变量有了一个默认值,静态成员方法有了一个地址值并且初始化完成,可以被调用,而非静态成员变量name和age则也会被加载,但不会初始化,非静态方法show01()也被加载,但没有初始化,有了一个方法区的地址,至此,.class文件的加载完成.

​ 程序继续运行,main方法加载到栈内存,在栈内存里开辟空间,我们通过类名访问静态成员变量和静态成员方法,这个过程将加载到栈内存,通过类名找到类名在方法区的地址,对静态成员变量和静态成员方法进行调用,由于已经初始化完成,调用静态成员变量将会输出对应数据类型的默认值,而调用静态成员方法则会直接将静态成员方法加载到栈内存,对静态成员变量university进行打印.

​ 程序继续往下,我们新建了一个StaticStudent对象student,该对象就会在堆内存里开辟空间,并对非静态成员变量和非静态成员方法进行初始化,最终返回堆内存地址给栈内存对象名,输出非静态成员变量值即通过地址访问堆内存获取其值,调用非静态成员方法,也是通过地址访问堆内存,并通过堆内存里的地址访问方法区,输出语句.

​ 最后main方法执行结束,从栈内存消失,堆内存对象等待jvm清理
